PostsController::removeImagesIfNecessary

Before editing a post, the post record is stored in the session.

This way, we know which images are linked to the post. Now that the post is edited and submitted, we need to make sure the posted images were in the original post. When it's not, somebody is trying to temper with it. When it's not _anymore_ it means the user has deleted the image while editing the post.
