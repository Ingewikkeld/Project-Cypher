\Pheanstalk_Pheanstalk

Pheanstalk is a pure PHP 5.2+ client for the beanstalkd workqueue.

The Pheanstalk class is a simple facade for the various underlying components.

  • Author: Paul Annesley

Methods

private

  • _dispatch() — Dispatches the specified command to the connection object.
  • _reconnect() — Creates a new connection object, based on the existing connection object, and re-establishes the used tube and watchlist.

public

  • __construct()
  • bury() — Puts a job into a 'buried' state, revived only by 'kick' command.
  • delete() — Permanently deletes a job.
  • ignore() — Remove the specified tube from the watchlist.
  • kick() — Kicks buried or delayed jobs into a 'ready' state.
  • listTubeUsed() — The name of the current tube used for publishing jobs to.
  • listTubes() — The names of all tubes on the server.
  • listTubesWatched() — The names of the tubes being watched, to reserve jobs from.
  • pauseTube() — Temporarily prevent jobs being reserved from the given tube.
  • peek() — Inspect a job in the system, regardless of what tube it is in.
  • peekBuried() — Inspect the next job in the list of buried jobs of the specified tube.
  • peekDelayed() — Inspect the shortest-remaining-delayed job in the specified tube. If no tube is specified, the currently used tube in used.
  • peekReady() — Inspect the next ready job in the specified tube. If no tube is specified, the currently used tube in used.
  • put() — Puts a job on the queue.
  • putInTube() — Puts a job on the queue using specified tube.
  • release() — Puts a reserved job back into the ready queue.
  • reserve() — Reserves/locks a ready job in a watched tube.
  • reserveFromTube() — Reserves/locks a ready job from the specified tube.
  • setConnection()
  • stats() — Gives statistical information about the beanstalkd system as a whole.
  • statsJob() — Gives statistical information about the specified job if it exists.
  • statsTube() — Gives statistical information about the specified tube if it exists.
  • touch() — Allows a worker to request more time to work on a job.
  • useTube() — Change to the specified tube name for publishing jobs to.
  • watch() — Add the specified tube to the watchlist, to reserve jobs from.
  • watchOnly() — Adds the specified tube to the watchlist, to reserve jobs from, and ignores any other tubes remaining on the watchlist.
